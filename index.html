Windows 下 Hexo个人博客的搭建与Github的部署

一、安装环境

  1. 以为Hexo需要Node.js进行支持，因此在电脑上需要安装Node.js环境,建议下载LTS(长期支持版)，官方建议使用 Node.js 12.0 及以上版本。Node.js下载地址

  2. 点击下载好的Node.js安装包进行安装，安装完毕后会有两个组件，一个是Node.js本身，另一个则是npm包管理器组件。

  3. 在控制台查看Node.js以及npm是否安装成功。

二、Hexo博客搭建

  1. 在命令行输入npm install hexo-cli -g进行Hexo的全局安装
  2. 在命令行输入hexo -v 检查Hexo安装状态

)

  1. 在电脑中选定一个合适的位置新建一个文件夹，这个文件夹会存储你博客中的所有东西，在命令行中进入这个新建文件夹，输入hexo init 进行初始化。

  1. 安装完毕后在控制台输入hexo s 在本地启动下Hexo，在浏览器输入http://localhost:4000/，查看Hexo默认页面。

  2. 创建一篇新的博文，hexo new "新博文名称" ，使用本地编辑器进行编辑。本地编辑器我是用的是Typora。Typora下载地址|国内镜像下载地址

  3. 完成博客编写后，先使用hexo clean 清理一下，再使用hexo g 进行文章的编译，最后使用hexo s 进行本地测试。

三、Hexo部署到Github

  1. 登录你自己的github账号，在右上角的加号下选择New repository,Repository name 名字必须遵循特定规则：github用户名.github.io，Description个人随意。点击进行创建。

  2. 在博客所在的文件夹下进入命令行界面，安装hexo-deployer-git插件.

  1. 插件安装完毕后，在博客文件夹中找到_config.yml文件打开。在最底部的Deployment内修改为如下：

  1. 在命令行输入hexo d 部署到远端，在过程中会让你输入Github的账号密码进行验证。在输入密码时是不回显输入的，输入完成后按回车即可。

  2. 推送完成后，刷新博客所在的github仓库，发现仓库中多了一堆文件即代表推送成功。

  3. 在浏览器输入仓库名程后即可查看部署到远端的博客。

  4. 部署完成后，新建的文章在本地预览无误后，使用hexo d 即可推送到远端。
